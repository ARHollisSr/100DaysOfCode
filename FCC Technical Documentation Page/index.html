Introduction

The joy of writing in JavaScript can also be impacted by the tools that you as a developer choose to use. There are a number of tools available to enhance your experience and improve the overall quality of your code, i.e., linters, typo correction, type-checking. Although a lot of these tools are stand-alone, their usefulness can be greatly compounded if included with your development editor.

Setting up Vim

For the sake of this text, the suggestion here is to use Neovim versus vanilla Vim. Vanilla Vim may not have all of the suggested plugin-ins which are available in Neovim.

Linting

From a high level, a linter looks at your code and reveals any potential errors without having to discover them at runtime. As of this writing, the most popular linter for JavaScript is ESLint; it supports modern JS features, JSX and it can be extended with additional features.

To get up an running with ESLint requires a few simple steps that are to be run from within your existing JavaScript project:

yarn add -D eslint yarn eslint -- --init # Or, if you're using npm npm install -D eslint ./node_modules/.bin/eslint --init

Code above: Installing ESLint via a project through Yarn or NPM

Ther are a number of VIM Linters but I would recommend using Ale. As it has some very useful features that differ from other offerings, such as the ability to run linters asynchronously and checking your files while you type without requiring a save.

Here are some other examples of using eslint.

$ eslint foo.js /path/to/foo.js 1:5 error 'foo' is assigned a value but never used no-unused-vars 3:10 error 'bar' is defined but never used no-unused-vars 5:1 error Expected indentation of 1 tab but found 2 spaces indent ✖ 3 problems (3 errors, 0 warnings) 2 errors, 0 warnings potentially fixable with the `--fix` option.

Above code: example of default eslint output

Here is another...

let g:ale_lint_on_enter = 0

Above code: Ensuring the linter doesn't run after opening a file

ESLint and Prettier

Most coding styles are developed over time and more times than not are a matter of personal choice. However, if you are working professionally, you may have to adhere to team guidelines regarding styles. To this effect, tools like Prettier exist which aims to alleivate friction amongst team members by simply producing pefectly formatted code.

Coupled with ESLint to which has its own methods for formatting code. You can use a two step approach to fixing code in Vim:

  1. Configure ESLint to run Prettier automatically
  2. Configure Vim to fix files through ESLint
This will allow Vim to reveal errors from ESLint and Prettier, basically fixing both at the same time.

Prettier through ESLint

The first thing we want to do is to get ESLint reporting on Prettier errors. As luck would have it, there is a plugin/configuration pair for this situation.

To install peform the following:

yarn add -D prettier eslint-plugin-prettier eslint-config-prettier # Or, if you're using npm npm install -D prettier eslint-plugin-prettier eslint-config-prettier

Code above: Installing prettier for eslint.sh

Now update your ESLint configuration to look like the following:

{ “extends”: [ “eslint:recommended”, “prettier” ], “plugins”: [ “prettier” ], “rules”: { “prettier/prettier”: “error” } }

Code above: .eslintrc

Now running ESLint will report issues from ESLint and Prettier and fixing ESLint errors will fix Prettier ones too.

Here is a list of some Prettier Plugins:

  • Python Prettier
  • Php Prettier
  • Swift Prettier

Here are few notable Eslint plugins:

  • eslint-cli - This is the eslint command that executes a local installed ESLint.
  • eslint-find-rules - Find built-in ESLint rules you don't have in your custom config
  • eslint-index - CLI for finding and managing rules in ESLint config files
Autocomplete

No JS enviornment is complete without a good autocomplete experience. Out of the box VIM has some basic functionality through omnicomplete. However, you can do better with tools like TypeScript and Flow.

The most recommeded plugin for a richer autocomplete experience is deoplete. Deoplete includes a framework for assiting in the autocompletion of data using Vim. Here are a few recommended plugins:

  • deoplete-ternjs - - Autocomplete powered by Tern
  • deoplete-flow - - Autocomplete powered by Flow
  • nvim-typescript -- Provides Deoplete suggestions and a host of other tools for TypeScript development
